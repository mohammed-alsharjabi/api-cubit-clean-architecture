# API Cubit Clean Architecture — Flutter Learning Project

A compact Flutter project created to practice API integration, Cubit state management, repository separation, token handling, and functional error modeling.

> **Project type:** Architecture and API learning project  
> **Status:** Educational reference project

## العربية

مشروع تدريبي ركزت فيه على تعلم وتنفيذ ربط الـAPI باستخدام Dio، إدارة الحالة عبر Cubit، فصل الـRepository، التعامل مع JWT، واستخدام Dartz لتمثيل النجاح والفشل.

## Topics Practiced

- REST API integration with Dio
- Cubit and immutable state flows
- Repository-based data access
- JWT decoding
- Local token persistence
- Image selection
- Functional success and failure handling with Dartz

## Tech Stack

- Flutter and Dart
- Flutter BLoC / Cubit
- Dio
- Dartz
- SharedPreferences
- JWT Decoder
- Image Picker

## Project Positioning

This repository is intentionally small. It is not presented as a full commercial product; its purpose is to demonstrate focused architectural and API-learning work from an earlier stage of my development journey.

## Running Locally

```bash
flutter pub get
flutter run
```

A reachable API endpoint and valid environment configuration may be required.
